BASH

BASH is one of several command line interfaces (CLI) to come out of the UNIX world. Most are still only used exclusively with UNIX or derivatives of UNIX, like Linux. For the most part, they were developed at universities in the early days of computing, before things like operating systems and CLIs were standardized. So one may have been created at Stanford, another at Berkeley, still another at MIT, etc. They all serve the same purpose (mostly file manipulation) and have similar commands.

Many CLIs are derivatives (maybe later generation is more descriptive, if less accurate) of a previous CLI and there are several with three or four (maybe more) such generations. Derivatives don't usually change the commands or syntax of the previous version (although they might) and instead add commands not supported in a previous incarnation.

Derivative CLIs are usually given a new name evocative of the original; SH begat BASH.

CLIs can seem daunting, with a veritable forest of commands to perform simple functions like naming files, and not always intuitive command names and nomenclature. It seems easier that it would be easier to point and click to access these sames functions from a GUI. Although, perhaps more intuitive, GUIs are less efficient to use. Hands and minds must shift from keyboard to mouse and interfaces must be navigated to find the functions required.

CLIs have a single interface -- the command line. All commands are valid input at all times. Simple functionality can be accessed directly by typing, usually, simple commands. CLIs are generally more efficient than GUIs.

Windows has it's own CLI that serves the same purpose as the various UNIX CLIs. It started out as the MS-DOS command line, which in turn was based on CP/M and to a lesser extent IBM's mainframe DOS. Over time it has adopted many of the commands found in UNIX CLIs, and there a number of commands specific to the Windows environment. Although there are versions of most UNIX CLIs ported to Windows, they aren't necessary nor all that much superior to MS-DOS. As such, they are somewhat rare and generally serve niche markets.

One of those markets is computer programming, especially those products and services that are cross-platform. Web browsers and the Internet are common examples. Both were originally developed in academic environments (where UNIX was far more prevalent than Windows) and both were originally and intentionally ignored by Microsoft.

As a result UNIX CLIs have found a place among programmers, even in a Windows environment.

Common BASH Commands
cd Change Directory

cd needs to find the specified directory and really only has two possible starting references: the current directory and the root directory of the file system.

Although you could type the fully-qualified paths and name of each file, every time, no one really wants to and a simple spelling mistake can invalidate the command causing it to fail or could result in moving the file to the wrong location, possibly causing it to be lost. Luckily, the file structure also maintains references to the current directory and its parent which make it possible to enter relative paths to the resource and shorten the file path considerably.

. the current directory
.. the parent directory
explorer Edit a file (Windows only)

Invokes the Windows launcher to open the indicated file with an appropriate editor of viewer. The application will be launched in a window.
mkdir Make a Directory
mv Move a file or directory
This is one of the less intuitive aspects of UNIX CLIs. BASH and most UNIX CLIs do not have a separate command to rename a file or directory. Instead, mv is used to do both.
rm Remove a file
rmdir Remove Directory
touch Create a file
